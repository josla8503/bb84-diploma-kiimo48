# BB84 

Учебная реализация протокола **BB84** (Bennett & Brassard, 1984) на Python.  
Программа демонстрирует полный цикл квантового обмена ключами: генерацию кубитов, передачу по квантовому каналу, обнаружение перехватчика (Евы), согласование ключа и финальное шифрование сообщения методом одноразового блокнота (OTP).

---

## Структура проекта

```
bb84_protocol.py      — Ядро протокола (кодирование/декодирование кубитов, просеивание, QBER)
channel.py            — Квантовый канал (шум, перехват Евы)
postprocessing.py     — Согласование ключа (Cascade-подобный алгоритм) и усиление секретности
main.py               — Точка входа, CLI, опциональная визуализация (dearpygui)
statistics_report.py  — Массовые эксперименты, таблицы и графики
visualizer_dpg.py     — GUI-визуализатор на DearPyGui (необязательный)
```

## Быстрый старт

### Графический интерфейс
Просто запустите главный файл без аргументов. Откроется интерактивный лаунчер:
```bash
python main.py
```

### Базовый запуск (без Евы, без сообщения)

```bash
python main.py demo
```

### Передать зашифрованное сообщение

```bash
python main.py demo --message "Привет, мир!"
```

### Включить перехватчика (Еву)

```bash
python main.py demo --eve --message "Секретный текст"
```

### Задать параметры вручную

```bash
python main.py demo \
  --n 500 \
  --p_err 0.03 \
  --eve \
  --p_intercept 0.6 \
  --message "Тест"
```

| Параметр        | По умолчанию | Описание                                              |
|-----------------|:------------:|-------------------------------------------------------|
| `--n`           | 200          | Количество кубитов                                    |
| `--p_err`       | 0.01         | Вероятность шума в канале (0.0 – 1.0)                 |
| `--eve`         | выкл.        | Активировать перехватчика                             |
| `--p_intercept` | 1.0          | Доля кубитов, перехватываемых Евой                    |
| `--message`     | —            | Текст для шифрования (OTP)                            |
| `--no_viz`      | выкл.        | Отключить GUI (только текстовый вывод)                |

## Статистические эксперименты

Файл `statistics_report.py` позволяет прогнать серию экспериментов по сетке параметров и получить CSV-таблицу и графики.

```python
# Пример — запустить из Python или дополнить своим скриптом:
from statistics_report import run_experiments, build_table, plot_key_rate, plot_eve_detection

param_grid = {
    "n":           [200, 500],
    "p_err":       [0.0, 0.02, 0.05],
    "eve_enabled": [False, True],
    "p_intercept": [0.5, 1.0],
}

results = run_experiments(param_grid, n_repeats=50)
build_table(results, csv_path="results.csv")
plot_key_rate(results, save_path="key_rate.png")
plot_eve_detection(results, save_path="eve_detection.png")
```

Результаты сохраняются в `results.csv`, графики — в PNG-файлы.

## Как работает алгоритм (краткое введение)

### Генерация кубитов (Алиса)

```
Бит × Базис → Поляризация фотона
  0  ×  0°   →   0°   (горизонталь)
  1  ×  0°   →  90°   (вертикаль)
  0  × 45°   →  45°   (диагональ)
  1  × 45°   → 135°   (антидиагональ)
```

Алиса случайно выбирает бит и базис (прямолинейный `0°` или диагональный `45°`) для каждого кубита.

### Передача и измерение (Боб)

Боб независимо выбирает базис для каждого кубита. Если базисы совпали — он получает правильный бит. Если нет — результат случаен.

```
Алиса: базис 0°,  кубит 90° → Боб: базис 0°  → бит = 1  ✓ (совпало)
Алиса: базис 0°,  кубит  0° → Боб: базис 45° → бит = ?  ✗ (не совпало, выброс)
```

### Просеивание ключа (Sifting)

Алиса и Боб публично сравнивают **только базисы** (не биты). Позиции, где базисы совпали (~50%), образуют «просеянный» ключ.

### Оценка уровня ошибок (QBER)

Из просеянного ключа случайно берётся ~25% позиций и сравниваются открыто. Результат — **QBER** (Quantum Bit Error Rate).

```
QBER без Евы и шума : ~0%
QBER с шумом канала : ~p_err
QBER при 100% перехвате Евой: ~25%  (Ева угадывает базис в 50% случаев)

Порог обнаружения: QBER > 11% → перехват обнаружен
```

### Согласование ключа (Error Reconciliation)

Используется алгоритм, схожий с **Cascade**: ключ разбивается на блоки, сравниваются биты чётности, ошибки исправляются методом бинарного поиска. «Утечка» информации (количество открыто переданных бит чётности) учитывается на следующем шаге.

### Усиление секретности (Privacy Amplification)

Финальный ключ сжимается до длины:

```
l = n × (1 − H(QBER) − leak_ec)
```

где `H(p) = −p·log₂(p) − (1−p)·log₂(1−p)` — бинарная энтропия. Это гарантирует, что любая информация, утёкшая к Еве, математически «размывается» через SHA-256.

### Шифрование OTP

Полученный ключ используется как **одноразовый блокнот**: каждый бит сообщения XOR-ится с соответствующим битом ключа. При правильном ключе это абсолютно стойкий шифр (теорема Шеннона, 1949).

```
Шифртекст = Сообщение XOR Ключ
Сообщение = Шифртекст XOR Ключ   (у Боба)
???        = Шифртекст XOR Мусор  (у Евы без ключа)
```
